# 明治神宮野球場 気温お知らせBot（Bluesky版）

明治神宮野球場の最寄り気象庁公式観測点（東京/大手町、地点番号44132）の気温を10分おきに監視し、
その日(JST)に初めて 25/30/35/40度 に到達したタイミングで **Bluesky** に自動投稿します。

完全無料・課金一切なしで実装できます（X/Twitterは2026年2月以降、投稿APIが有料化されたため非採用）。

## 投稿文言

| 気温帯 | 投稿文言 |
|---|---|
| 25℃以上30℃未満 | 夏日よ |
| 30℃以上35℃未満 | 真夏日よ |
| 35℃以上40℃未満 | 猛暑日よ |
| 40℃以上 | 酷暑日よ |

同日中に既により高い（または同じ）レベルを投稿済みの場合は再投稿しません。

## セットアップ手順（人力で行う作業）

### 1. Blueskyアカウントを新規作成する
1. https://bsky.app/ にアクセスし、新規アカウントを作成（メールアドレスのみでOK、電話番号不要）
2. 好きなハンドル名を設定（例：`manatsubiyo.bsky.social`）
3. プロフィールに自動投稿botであることを明記しておくと親切です
   - 例：「⚾明治神宮野球場周辺の気温を自動でお知らせするbotです／出典：気象庁アメダス」

### 2. アプリパスワードを発行する（開発者登録は不要）
Blueskyは X と異なり、開発者ポータルへの登録・審査・課金は一切不要です。普段のパスワードとは別の
「アプリ専用パスワード」を発行するだけでAPIが使えます。

1. Bluesky にログインした状態で **設定（Settings）→ Privacy and Security → App Passwords** を開く
2. 「Add App Password」をクリックし、名前を入力（例：`temp-bot`）して発行
3. 表示された文字列（例：`xxxx-xxxx-xxxx-xxxx`）をその場でコピーして控える
   - ※この画面を閉じると二度と表示されないので注意

### 3. GitHubリポジトリを準備する
1. GitHubアカウントを作成（未所持の場合）
2. 新規リポジトリを作成（**Public** にすると Actions の実行時間が無料無制限になります）
3. このフォルダの中身（`check_and_post.py`, `requirements.txt`, `state.json`, `.github/workflows/check_temp.yml`）を
   そのままリポジトリ直下にアップロード（GitHub Web UI の「Add file → Upload files」でドラッグ＆ドロップ可）

### 4. シークレットを登録する
リポジトリの **Settings → Secrets and variables → Actions → New repository secret** から、以下2つを登録
- `BLUESKY_IDENTIFIER` … Blueskyのハンドル名（例：`manatsubiyo.bsky.social`）またはログイン用メールアドレス
- `BLUESKY_APP_PASSWORD` … 手順2で発行したアプリパスワード

### 5. Actionsの書き込み権限を有効化する
**Settings → Actions → General → Workflow permissions** で
「**Read and write permissions**」を選択して保存（state.json をBotがコミットし直すために必須）

### 6. 動作確認
1. リポジトリの **Actions** タブ → 「Check Temperature and Post to Bluesky」を選択
2. 「Run workflow」ボタンで手動実行
3. ログを確認し、気温取得・投稿判定が正しく動いているか確認
   （閾値未達なら投稿はされず、ログにその旨が出力されます）

これで完了です。以降は10分おきに自動でチェックし、条件を満たした時だけBlueskyに投稿されます。

## 運用上の注意

- **気温の取得元**：気象庁アメダス「東京（大手町）」。神宮球場の厳密な現地気温ではなく、最寄りの公式観測点の値です。出典として気象庁データを利用している旨をプロフィール等に明記することを推奨します。
- **GitHub Actions の cron 精度**：GitHubの仕様上、混雑時は数分程度の実行遅延が起こることがあります（高頻度cronは公式にも「正確な時刻を保証しない」と明記されています）。
- **コスト**：Bluesky APIは投稿・読み取りとも無料です。GitHub Actionsもパブリックリポジトリなら無料無制限です。完全無料で運用できます。
- **投稿文言の拡張**：現状は指定された文言（「夏日よ」等）のみを投稿する設計です。気温や日付、出典などを併記したい場合は `check_and_post.py` の `text_for_level` 呼び出し部分を変更してください。
- 取得失敗（気象庁側の欠測・通信エラー等）の場合はスキップし、次回（10分後）に再試行します。
- アプリパスワードは漏洩するとアカウントの投稿権限を奪われる可能性があるため、GitHub Secrets以外の場所に書き残さないようにしてください。
